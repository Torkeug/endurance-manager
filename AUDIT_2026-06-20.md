## 1. Security Findings

### CRITICAL

**SEC-1 ‚Äî `app/api/register-driver/route.js`**  
Endpoint accepts `auth_user_id` from POST body and inserts it via service role with no check that the caller's session matches the UUID. Any logged-in user can create a `drivers` row linked to an arbitrary user's account.  
**Fix:** Call `supabase.auth.getUser()` from the request cookie and verify `user.id === auth_user_id` before inserting.

---

### HIGH

**SEC-2 ‚Äî `app/api/notify-admins/route.js`, `notify-driver-approved/route.js`, `notify-admins-approval/route.js`, `notify-stale-sync/route.js`**  
All four email-sending endpoints are completely unauthenticated. Anyone on the internet can POST to trigger Resend emails. `notify-stale-sync` also accepts a caller-controlled `driver_email`, making it an open email relay.  
**Fix:** Require `isAdmin()` session on admin-notification routes. Protect `notify-stale-sync` with `CRON_SECRET` bearer token, or make it an internal function called directly from the cron route.

**SEC-3 ‚Äî `app/api/iracing/lap/route.js:9`, `app/api/iracing/event/route.js:8`, `app/api/cron/cleanup-iracing-laps/route.js:6`, `app/api/cron/check-stale-syncs/route.js`**  
All four use `if (KEY && authHeader !== ...)` or `if (authHeader !== \`Bearer ${KEY}\`)` ‚Äî fails open when the env var is unset (becomes `"Bearer undefined"`, exploitable by sending that exact string). The cleanup route would allow any caller to delete all lap rows older than 7 days.  
**Fix:** Change all to `if (!KEY || authHeader !== \`Bearer ${KEY}\`)` ‚Äî fail closed when env var absent.

**SEC-4 ‚Äî `app/api/garage61-sync/route.js`**  
`driver_id` query param is not verified against the requester's identity or team membership. Any authenticated user can fetch another driver's Garage61 laps by DB ID (IDOR).  
**Fix:** Verify requesting driver shares a team entry with the target, or restrict to admins.

---

### MEDIUM

**SEC-5 ‚Äî `app/api/auth/iracing/route.js`**  
`?mode=syncall` is not admin-gated at initiation. The admin check only fires in the callback after the iRacing token has already been exchanged.  
**Fix:** Add `isAdmin()` check in the initiation route for `mode === "syncall"`.

**SEC-6 ‚Äî `app/api/auth/callback/garage61/route.js:59`**  
`console.error("Garage61 token response missing access_token:", tokenData)` logs the full token response, potentially including `refresh_token`, to Vercel runtime logs.  
**Fix:** Log `tokenData?.error` or `Object.keys(tokenData)` only.

**SEC-7 ‚Äî `app/api/auth/iracing/route.js:26`, `app/api/auth/garage61/route.js:22`**  
`returnTo` is only sanitized by stripping `|`. A value like `//evil.com/path` could be treated as a protocol-relative URL by browsers (open redirect).  
**Fix:** Validate `returnTo` matches `^/[^/]`; reject or default to `/pilotes` otherwise.

**SEC-8 ‚Äî `app/api/notify-admins/route.js:50`, `notify-admins-approval/route.js:40`**  
`newDriverName`, `newDriverEmail`, `driverName`, `approvedByName` interpolated directly into HTML email bodies with no escaping. Enables link/HTML injection into admin inboxes.  
**Fix:** Run all user-supplied values through an HTML escape function (`&`, `<`, `>`, `"`, `'`) before interpolation.

---

### LOW / MEDIUM

**SEC-9 ‚Äî `app/api/garage61-laps/route.js`, `garage61-practice/route.js`, `garage61-sync/route.js`**  
Auth checks only verify a valid Supabase session, not `approved = true`. Users stuck at `/pending` can call Garage61 endpoints.  
**Fix:** After fetching `requestingDriver`, add `if (!requestingDriver?.approved) return 403`.

## 2. Code Quality & Bug Findings

### State lock-up bugs (StintGrid.js)

**BUG-1 ‚Äî `StintGrid.js:1838` ‚Äî `updateStint` never resets `saving` on DB error**  
No try/finally. A failed update leaves the stint visually dimmed forever and `savingRef` stuck ‚Äî silently drops all realtime updates for that stint until page reload.  
**Fix:** Wrap the await in try/finally, call `setSaving(null)` in the finally block.

**BUG-2 ‚Äî `StintGrid.js:1855` ‚Äî `updateActualEnd` same issue**  
Same pattern, same consequence as BUG-1.

**BUG-3 ‚Äî `StintGrid.js:1699` ‚Äî `handleCreateStrategy` doesn't reset `creatingStrategy.current` on error**  
If the stints insert fails, `creatingStrategy.current` stays `true` ‚Äî strategy creation is permanently disabled for the rest of the session.  
**Fix:** Reset `creatingStrategy.current = false` in a finally block.

---

### Logic bugs

**BUG-4 ‚Äî `StintGrid.js:476` ‚Äî `_isLastStint` uses `find()` instead of `findLast()`**  
When multiple stints extend past `raceEnd`, `find()` picks the first one, not the actual last stint. That first stint gets its laps trimmed incorrectly; the real final stint goes uncorrected.  
**Fix:** Replace `.find()` with `.findLast()` for the covering-stint detection.

**BUG-5 ‚Äî `app/api/cron/check-stale-syncs/route.js:43` ‚Äî `stale_sync_notified_at` written before email confirmed**  
The `fetch()` to `notify-stale-sync` is fire-and-forget (no `await`). If the email fails, the 30-day cooldown is stamped anyway.  
**Fix:** `await` the notify call; only write `stale_sync_notified_at` if `res.ok`.

**BUG-6 ‚Äî `app/api/cron/check-stale-syncs/route.js:15` ‚Äî query includes drivers who never linked iRacing**  
`.or('last_driver_sync_at.is.null,...')` catches drivers with `iracing_id = null`. They receive stale-sync emails indefinitely.  
**Fix:** Add `.not('iracing_id', 'is', null)` to the query.

**BUG-7 ‚Äî `StintGrid.js:1266` ‚Äî parallel fetch swallows errors, loading spinner never clears**  
`Promise.all` errors are not caught. If strategies or stints fail to load, `setLoading(false)` is never called ‚Äî infinite spinner.  
**Fix:** Add `.catch(err => { console.error(err); setLoading(false); })`.

**BUG-8 ‚Äî `StintGrid.js:469` ‚Äî `effectiveEnd` with `irl_end_actual` set but `_irlEnd` null**  
If a stint has `irl_end_actual` stamped but the calculated `_irlEnd` is null (data inconsistency), `currentTime` advances to actual end + pit-stop, potentially producing future timestamps for already-completed stints and corrupting the chain downstream.

**BUG-9 ‚Äî `RaceMode.js:426` ‚Äî `iracing_events` may lack RLS, exposing all teams' events**  
The realtime subscription on `iracing_events` uses the browser anon client. If no RLS policy restricts rows by team, every user in realtime sees every team's events. The API route at `/api/iracing/event` uses service role (no RLS), so the insert side is fine ‚Äî but the read side deserves verification.  
**Fix:** Verify RLS policy exists on `iracing_events` limiting reads to the driver's team entries.

---

### React hook errors (beyond ESLint ‚Äî logic impact)

**HOOK-1 ‚Äî `StintGrid.js:1413` ‚Äî `JSON.stringify(driverIds)` in dep array causes unnecessary effect re-runs**  
`driverIds` is recomputed every render; `JSON.stringify` creates a new string each time even when contents haven't changed.  
**Fix:** Stabilise `driverIds` with `useMemo`.

**HOOK-2 ‚Äî `StintGrid.js:1451` ‚Äî strategy-switch effect fires when tab is hidden**  
`[selectedStrategyId, isActive]` triggers a DB fetch even when `isActive === false`.  
**Fix:** Add `if (!isActive) return;` at the top of the effect.

**HOOK-3 ‚Äî `StintGrid.js:1550` ‚Äî `autoOpenRecalc` effect missing `openRecalcModal` in deps**  
If `driverPerf` changes between renders, the effect could call a stale `openRecalcModal` with outdated perf data.  
**Fix:** Wrap `openRecalcModal` in `useCallback` or add it to the deps array.

**HOOK-4 ‚Äî `StintGrid.js:599` ‚Äî `isStintCompleted` calls `new Date()` instead of memoized `now` state**  
Creates subtle inconsistency between the active-stint highlight and completed/eligible checks.

**HOOK-5 ‚Äî `StintGrid.js:1346` ‚Äî strategy resolution can flicker on fast tab switch**  
Async fetch may complete after `selectedStrategyId` has changed, causing `resolvedStratId` to fall back to the active strategy rather than the user's current selection.

---

### Minor / lower priority

**MINOR-1 ‚Äî `lib/garage61.js:69` ‚Äî 401 retry drops request options**  
The retry after token refresh only forwards the `Authorization` header, losing `method`, `body`, `signal`. Harmless for current GET-only callers, but a footgun for any future POST through `g61Fetch`.

**MINOR-2 ‚Äî `RaceMode.js:822` ‚Äî `markPitStop` uses flat `refuel_time_seconds`, ignoring per-litre rate**  
Pit exit countdown is wrong during race for entries using variable refuel rates. StintGrid corrects it on next open, but live race view shows the wrong value.

**MINOR-3 ‚Äî `RaceMode.js:410` ‚Äî `fetchLiveFuel` magic numbers `< 3` and `limit=5` in two places**  
The guard threshold and the query limit are separate magic numbers; changing one silently breaks the other. Extract as named constants.

**MINOR-4 ‚Äî `RaceMode.js:868` ‚Äî `undoLastPit` stale `last` used as fallback**  
`last` is found before the confirm modal opens. By confirmation time, a realtime update may have changed which stint is last-stamped. The code re-derives `freshLast` from `stintsRef.current` at line 884 but falls back to the stale `last` at line 885 (`?? last`).

**MINOR-5 ‚Äî `lib/supabase-browser.js:17` ‚Äî module-level singleton created at import time**  
`supabaseBrowser = createBrowser()` runs at import, including on the server if this module is accidentally imported in a server context. Current "use client" placement prevents this, but there's no runtime guard.

**MINOR-6 ‚Äî `StintGrid.js:1529` ‚Äî `JSON.stringify(calculated)` in persist effect dep array**  
Wasteful O(n stints) serialisation on every render cycle.

**MINOR-7 ‚Äî `app/api/garage61-laps/route.js:41` ‚Äî `driver_slug` not validated**  
The slug comes from the query string and is never checked to be a non-empty string of expected shape. A null/unusual value returns `{ lap: null }` silently with no error surfaced to the caller.

**Common fix for `set-state-in-effect` (localStorage theme pattern):**  
Replace:
```js
const [theme, setTheme] = useState("dark");
useEffect(() => { setTheme(localStorage.getItem("theme") || "dark"); }, []);
```
With lazy initializer:
```js
const [theme, setTheme] = useState(() =>
  typeof window !== "undefined" ? localStorage.getItem("theme") || "dark" : "dark"
);
```

### CRITICAL

**SEC-10 ‚Äî `app/auth/iracing/route.js:41` ‚Äî OAuth CSRF: PKCE verifier in `state` URL param**  
The state parameter is `${verifier}|${mode}|${returnTo}`. Transmitting the PKCE verifier in-band in the redirect URL means an attacker can forge a state string pairing any code with a crafted verifier ‚Äî CSRF protection is absent.  
**Fix:** Generate a random nonce as the state value; store `{nonce ‚Üí verifier, mode, returnTo}` in a `SameSite=Strict; HttpOnly` cookie at initiation. In the callback, read the cookie, verify `state === nonce`, and reject on mismatch.

**SEC-11 ‚Äî `app/auth/garage61/route.js:37` ‚Äî OAuth CSRF: identical issue**  
State is `${verifier}|${returnTo}` ‚Äî same problem, same fix as SEC-10.

### HIGH

**SEC-12 ‚Äî `app/auth/callback/iracing/route.js:283` ‚Äî OAuth callback never validates state nonce**  
`const [verifier, mode, returnTo] = state.split("|")` ‚Äî state is parsed and trusted without verifying it was server-issued. Any request with a crafted state string and a valid auth code will be processed. The admin check for `syncall` runs after the token exchange, so replay is possible.  
**Fix:** Validate the nonce cookie before processing the code; reject immediately if missing or mismatched.

**SEC-13 ‚Äî `app/auth/callback/garage61/route.js:16` ‚Äî same**  
`const [verifier, returnTo] = (state || "").split("|")` ‚Äî no nonce validation.  
**Fix:** Same as SEC-12.

**SEC-14 ‚Äî `app/api/notify-stale-sync/route.js:9` ‚Äî unauthenticated + href injection**  
`driver_name`, `driver_email`, and `profile_url` come entirely from the caller's body. `profileUrl` is embedded unescaped in the email HTML as `<a href="${profileUrl}">`. An attacker can send phishing emails to arbitrary addresses with a crafted `href` (including `javascript:` or external URLs). This is distinct from SEC-2 (general lack of auth) and SEC-8 (name/email HTML injection) ‚Äî this one is a caller-controlled href.

No auth. Caller supplies a `driver_id` UUID; the route fetches and emails that driver. Any caller can probe UUIDs to discover valid driver emails, and trigger approval spam.

**SEC-16 ‚Äî `app/evenements/[id]/inscription/page.js:783,797` ‚Äî IDOR: signup on behalf of any driver**

**SEC-17 ‚Äî `app/auth/callback/iracing/route.js:405` ‚Äî syncall fetches wrong driver set**
